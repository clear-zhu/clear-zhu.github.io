Jpa生成sql语句的原理

PA在这里遵循Convention over configuration（约定大约配置）的原则，遵循spring 以及JPQL定义的方法命名。
这套机制会把方法名首先过滤一些关键字，比如 find…By, read…By, query…By, count…By 和 get…By 。系统会根据关键字将命名解析成2个子语句，第一个 By 是区分这两个子语句的关键词。这个 By 之前的子语句是查询子语句（指明返回要查询的对象），后面的部分是条件子语句。

JAVA学习

整体命名规则

1.包名是由小写的字母序列组成
2.类名大写字母开头，其他使用小写字母
3.类的实例是小写字母开头，后面的用大写字母开头
4.常量名称使用大写，并且写出完整的含义
5.参数的名称无具体规则。
6.数组的命名使用，类型+[]+数组名的形式。如：

编译的习惯：

1.程序最开始编写导入包和类语句，即 import 语句。import 语句可以有多行，编写完 import 语句后空一行。
2.定义 public 类，顶格书写。类的主体左括号“{”不换行书写，右括号“}”顶格书写。
3.定义 public 类中的变量，缩进书写。
4.定义方法用缩进书写，方法的左括号“{”不换行书写，右括号“}”和方法首行第一个字符对齐。方法体要再次缩进书写，最后一个变量定义和第一个方法定义之间、方法和方法之间最好空一行。

其中，new Person（），是一个对象，a是具体的实例，b属于引用。三者分别存放在堆，常量池和栈空间中。

小细节

1、Java的主类名称需要和整个项目名称一致，不然会报错，在生成主类时会自动生成项目名称。主类是public的，有且只有一个。

2、如何定义类方法属性（公开，保护，私有）

3、在定义类的函数时，static关键字指明该变量是类共享成员。

4、final关键字，可以修饰类，方法，成员变量。当修饰类时不能被继承，该类中的方法也都是final的。当修饰方法时，子类继承不能覆盖该方法。修饰常量时，该常量只能被赋值一次，便不可以再修改。

5、当子类继承父类的时候使用extends，调用父类中的方法使用super.XXX。
6.在java中的常量默认是占64位的双精度，在不需要高精度的情况下，定义为float的单精度类型。

参照博客：https://cloud.tencent.com/developer/article/1705945

Maven

Maven介绍

Maven是一个项目的管理器工具，可以用来构建工程，管理jar包，编译代码，自动化测试单元，打包，跨平台，生成报表，项目部署生成web站点等优点及功能。

Maven的依赖管理

Maven的依赖管理是利用pom.xml这个文档来查找项目所需要的jar包，在pom.xml中包含了所需要包的坐标位置。从而避免了直接将jar包引用进来，减小了整体项目的大小。
在pom.xml文件中查找包的坐标后，利用索引的方式来查找项目所需要的包，从而减小先得到坐标再找包的时间。

jar包仓库

在pom.xml中得到的坐标，就要去仓库去找项目所需的jar包。jar包仓库可以分为三类：本地仓库，第三方仓库，中央仓库。

本地仓库：用来存储从远程仓库或者中央仓库下载的插件和jar包，一般本地仓库的默认位置为：usr.dir/.m2/repository

第三方仓库：又称之为内部中心仓库，一般为公司创建，减少外部访问和下载的频率。

中央仓库：Maven软件中内置一个远程仓库地址：http://repo1.maven.org/maven2，它就是中央仓库，服务整个网络，Maven自己维护，里面存储了大部分流行的开源项目构件。

查找顺序：本地仓库->第三方仓库->中央仓库

Maven常用命令

compile：编译命令。将mian/java下的文件进行编译并输出到target下。

test：测试命令。执行test/java下的测试单元。

clean:清理命令。会删除target下的所有内容。

package打包命令。将java工程打包成jar包，web工程打包成war。

install：安装命令。将Maven打成jar包或者war包后发布到本地仓库。

Maven的工作周期

Clean Lifecycle：构建前进行清理作用。

Default Lifecycle：构建的核心部分，编译，测试，打包，部署等。

Site Lifecycle:生成项目报告，发布站点。

整体的工作流程是：
清理->编译->测试->报告->打包->部署
