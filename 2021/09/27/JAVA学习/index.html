整体命名规则

1.包名是由小写的字母序列组成
2.类名大写字母开头，其他使用小写字母
3.类的实例是小写字母开头，后面的用大写字母开头
4.常量名称使用大写，并且写出完整的含义
5.参数的名称无具体规则。

编译的习惯：

1.程序最开始编写导入包和类语句，即 import 语句。import 语句可以有多行，编写完 import 语句后空一行。
2.定义 public 类，顶格书写。类的主体左括号“{”不换行书写，右括号“}”顶格书写。
3.定义 public 类中的变量，缩进书写。
4.定义方法用缩进书写，方法的左括号“{”不换行书写，右括号“}”和方法首行第一个字符对齐。方法体要再次缩进书写，最后一个变量定义和第一个方法定义之间、方法和方法之间最好空一行。

小细节

1、Java的主类名称需要和整个项目名称一致，不然会报错，在生成主类时会自动生成项目名称。主类是public的，有且只有一个。

2、如何定义类方法属性（公开，保护，私有）

3、在定义类的函数时，static关键字指明该变量是类共享成员。

4、final关键字，可以修饰类，方法，成员变量。当修饰类时不能被继承，该类中的方法也都是final的。当修饰方法时，子类继承不能覆盖该方法。修饰常量时，该常量只能被赋值一次，便不可以再修改。

5、当子类继承父类的时候使用extends，调用父类中的方法使用super.XXX。
6.在java中的常量默认是占64位的双精度，在不需要高精度的情况下，定义为float的单精度类型。
